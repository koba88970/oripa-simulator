# 仮装景品の使い方

## 📝 仮装景品とは？

仮装景品は、まだ実際に仕入れていない景品を「仮登録」して、オリパの構成を検討するための機能です。

## ✅ 正しい使い方（2通り）

### パターン1：先に値を入力してからチェック

1. **景品名を入力**（例: 仮想レアカード）
2. **等級を選択**（例: 1等）
3. **ポイント還元価値を入力**（例: 70,000）← ここで入力
4. **仮装景品にチェック✓** ← チェックすると仕入れ価格に自動コピーされます
5. 「仕入れ価格（仮装景品）」フィールドに自動的に同じ値（70,000）が入力されます

### パターン2：チェックしてから値を入力

1. **景品名を入力**（例: 仮想レアカード）
2. **等級を選択**（例: 1等）
3. **仮装景品にチェック✓** ← 先にチェック
4. **ポイント還元価値を入力**（例: 70,000）← リアルタイムで仕入れ価格にも自動入力されます
5. 入力と同時に「仕入れ価格（仮装景品）」に同じ値が自動反映されます

## 🔍 動作確認方法

ブラウザのコンソール（F12キー → Consoleタブ）で以下のメッセージを確認できます：

### 仮装景品チェックをONにした時
```
🔄 仮装景品モードON
✅ ポイント還元価値を仕入れ価格にコピーしました: 70,000
```

### ポイント還元価値を入力した時
```
📝 ポイント還元価値が変更されました: 70,000 / 仮装景品モード: true
✅ 仕入れ価格に自動コピーしました: 70,000
```

### 景品を追加した時
```
📦 仮装景品を追加: {name: "仮想レアカード", purchasePrice: 70000, value: 70000, inputValue: "70,000"}
```

## ⚠️ よくある間違い

### ❌ 間違い1：仮装景品モードで「マスターから選択」を使おうとする
仮装景品モードでは、「📋 マスターから選択」ボタンは無効化されます。
仮装景品は実在しない景品なので、マスターから選ぶ必要はありません。

### ❌ 間違い2：仕入れ価格を手動で入力しようとする
仕入れ価格は自動的にポイント還元価値と同じ値になります。
手動で変更したい場合は、コピーされた後に変更できます。

### ❌ 間違い3：通常景品と同じように登録しようとする
通常景品：マスターから選択 → 必須
仮装景品：マスターから選択 → 不要（自動的にボタンが無効化）

## 💡 仕入れ価格が0になってしまう場合

もし仕入れ価格が0になってしまった場合、以下を確認してください：

1. **ブラウザのキャッシュをクリア**
   - Mac: Cmd + Shift + R
   - Windows: Ctrl + Shift + R

2. **正しい起動方法を使用**
   - `start-server.command`（Mac）または `start-server.bat`（Windows）を使用
   - `http://localhost:8000` で開く
   - `file://` で直接開かない

3. **コンソールでエラーを確認**
   - F12キーを押してConsoleタブを開く
   - 赤いエラーメッセージがないか確認

## 📋 仮装景品の利点

- ✅ 実際に仕入れる前にオリパ構成を検討できる
- ✅ 利益率やリスクを事前にシミュレーションできる
- ✅ 複数パターンを比較検討できる
- ✅ 仮装景品マスターに自動保存される

## 🎯 実際の使用例

### シナリオ：新しいオリパを企画中

1. まず「📝 仮オリパ（仮装景品で構成を検討）」にチェック
2. 仮装景品で構成を作成：
   - 1等：仮想PSA10カード（70,000円）
   - 2等：仮想レアカード（30,000円）
   - 3等：仮想ノーマルカード（5,000円）
3. 計算して利益率を確認
4. 良ければクラウドに保存
5. 実際に景品を仕入れたら、通常景品で本番のオリパを作成
